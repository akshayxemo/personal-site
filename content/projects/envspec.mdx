## Overview

EnvSpec is an open-source CLI tool that introduces schema-driven environment variable management for modern applications.

The tool helps developers validate configuration, generate environment templates, infer schemas from existing projects, and prevent deployment failures caused by missing or incorrectly configured environment variables.

Instead of relying on undocumented `.env` files, EnvSpec provides a version-controlled configuration contract while keeping secrets secure and local.

## Key Features

### Environment Validation

- Validate required environment variables before application startup
- Enforce type-safe configuration values
- Support enums, defaults, and validation rules
- Prevent runtime failures caused by invalid configuration

### Schema-Driven Workflows

- Define environment contracts using JSON schemas
- Standardize configuration across development, staging, and production
- Version-control environment requirements

### Environment Generation

- Generate `.env.example` and template files
- Improve onboarding for new developers
- Reduce manual configuration effort

### Schema Inference

- Generate schemas from existing `.env` files
- Accelerate migration of legacy projects
- Automatically infer supported data types

### Security & Git Protection

- Protect sensitive environment files
- Update `.gitignore` rules automatically
- Reduce accidental secret exposure
