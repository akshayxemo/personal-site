Atlas is the design system behind Northwind's product suite. It unifies five separate front-end codebases behind a single, versioned source of truth for tokens, components, and interaction patterns.

## The problem

Before Atlas, each product team shipped its own buttons, modals, and form controls. The result was visual drift, duplicated accessibility bugs, and a painful upgrade story whenever brand or platform requirements changed.

## Architecture

The system is built in three layers that compose cleanly:

- **Tokens** — a platform-agnostic JSON source compiled to CSS variables and TypeScript types.
- **Primitives** — unstyled, accessible behavior from Radix UI.
- **Components** — themed, opinionated React components consumed by product teams.

```ts title="theme.ts"
import { defineTokens } from "@atlas/tokens"

export const theme = defineTokens({
  color: {
    primary: { value: "oklch(0.54 0.21 285)" },
    surface: { value: "oklch(0.99 0.003 255)" },
  },
  radius: { md: { value: "0.625rem" } },
})
```

<Callout type="tip">
  Every token ships with a generated TypeScript type, so a typo in a color name becomes a compile-time error rather than a runtime surprise.
</Callout>

## Results

- Adopted across **6 product teams** and 200+ screens.
- Cut net-new UI code by an estimated **35%** through reuse.
- Established a single accessibility baseline audited against WCAG 2.1 AA.

## What I'd do differently

I would invest in visual regression testing earlier. We added it after the first major release, and retrofitting snapshots across hundreds of stories was more expensive than it needed to be.
