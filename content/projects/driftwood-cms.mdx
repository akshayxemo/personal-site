Driftwood is a content management layer that stores everything as MDX in a git repository while giving editors a clean, WYSIWYG-style authoring experience.

## The bet

Developers love git-based content — diffs, history, and code review for free. Editors hate it. Driftwood bridges that gap with a browser editor that commits on their behalf.

```ts title="commit.ts"
await octokit.repos.createOrUpdateFileContents({
  owner,
  repo,
  path: `content/${slug}.mdx`,
  message: `content: update ${slug}`,
  content: Buffer.from(body).toString("base64"),
})
```

## Lessons learned

- Real-time collaboration on top of git is genuinely hard — we scoped it to single-editor-per-document.
- Editors care far more about preview fidelity than about the underlying storage model.

<Callout type="warning">
  This project is archived. The hosted-CMS landscape matured quickly, and maintaining a git abstraction stopped being worth it for the team.
</Callout>
