Relay sits in front of a fleet of microservices and presents a single, strongly-typed graph to client teams — regardless of how each upstream service speaks.

## The idea

Client teams shouldn't need to know whether data comes from a REST endpoint, a GraphQL service, or a gRPC method. Relay normalizes all of it.

```ts title="resolver.ts"
export const userResolver = defineResolver({
  query: {
    user: async (_, { id }, ctx) => {
      const user = await ctx.grpc.users.get({ id })
      return mapUser(user)
    },
  },
})
```

## Features

- Schema stitching across heterogeneous backends.
- Per-field caching with Redis and automatic invalidation.
- Request-level tracing propagated to every upstream call.

<Callout type="warning">
  Gateways centralize risk. We invested heavily in load-shedding and circuit breakers so one slow upstream couldn't take down the whole graph.
</Callout>
